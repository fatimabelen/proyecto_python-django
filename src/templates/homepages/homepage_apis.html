Documentación de las APIS

Endpoint: /api/{entidad}/

Descripción: Este endpoint devuelve una lista con información resumida de todos los registros del sistema.
Métodos HTTP permitidos: GET
Parámetros de solicitud: No se requieren parámetros.
Códigos de estado en caso de éxito: 200 (OK)
Códigos de estado en caso de error: 404 (Not Found), 500 (Internal Server Error)
Formato de respuesta: JSON
Ejemplo: 


        [
          {
            "id": 1,
            "nombre": "María",
          },
          {
            "id": 2,
            "nombre": "Juan",
          }
        ]

Endpoint: /api/{entidad}/{id}/

Descripción: Este endpoint devuelve la información completa de un registro específico.
Métodos HTTP permitidos: GET
Parámetros de solicitud: Identificador (ID) del registro que se desea recuperar.
Códigos de estado: 200 (OK), 404 (Not Found), 500 (Internal Server Error)
Formato de respuesta: JSON
Ejemplo: 


        [
         {
            "id": 1,
            "nombre": "María",
            "apellido": "Perez",
            "activo": True
          }
        ]
